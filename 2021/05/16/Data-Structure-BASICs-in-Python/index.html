ADT: Abstract Data Type

  • stack, queue, deque, and list

    Stacks, queues, deques, and lists are examples of data collections whose items are ordered depending on how they are added or removed.

    • Stack: LIFO, last-in first-out

    For stack, the order that they are removed is exactly the reverse of the order that they were placed. So … stacks are fundamentally important, as they can be used to reverse the order of items.

    Items are added to and removed from one end called the ‘top’

    [Implementation]: Use a list to implement a stack, the end of the list will hold the top element of the stack.

    • Queue: FIFO, first-in first-out

    A queue is an ordered collection of items where the addition of new items happens at one end, called the “rear,” and the removal of existing items occurs at the other end, commonly called the “front”.

    Items are added to and removed from two ends called the “rear” (add items) and “front” (remove items).

    [Implementation]: Use a list to implement a queue, and we need to decide which end of the list to use as the rear and which to use as the front. This means that enqueue will be 𝑂(𝑛) and dequeue will be 𝑂(1).

    [Application]: Hot potato

    • Deque: a double-ended queue, two ends, a front and a rear. New items can be added/removed at either the front or the rear. Combine the capabilities of stacks and queues. Don’t require LIFO and FIFO orderings, it’s up to you to make a consistant use of the addition and removal operations.

    [Application]: Palindrome Checker

    • Unordered List: using Linked Lists to implement

    We need to be sure that we can maintain the relative positioning of the items. However, there is no requirement that we maintain that positioning in contiguous memory. — Linked Lists

    The basic building block for the linked list is the node, which must contain two pieces of information:

    • data field: the list item itself
    • reference: reference to the next node

    The linked list structure provides us with only one entry point, the head of the list. All of the other nodes can only be reached by accessing the first node and then following next links.
